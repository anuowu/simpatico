282  /*
283  * Implementation Notes:
284  *
285  * The pair list is stored in Atom* pointer arrays atom1Ptrs_, atom2Ptrs_,
286  * and in an integer array first_. Each element of atom1Ptrs_ contains a
287  * pointer to the first atom in a pair (the primary Atom). Each element
288  * of atom2Ptrs_ contains a pointer to the second atom in a pair (the
289  * secondary Atom). Secondary atoms that are neighbors of the same
290  * primary atom are listed consecutively. Element first_[i] contains the
291  * array index of the first element in atom2Ptrs_ that contains a neighbor
292  * of primary atom atom1Ptrs_[i]. Pointers to neighbors of primary atom
293  * *atom1Ptrs_[i] are thus stored in elements first_[i] <= j < first_[i+1]
294  * of atom2Ptrs_. Each pair is included only once, by including only
295  * pairs in which atom2Ptrs_[j]->id() > atom1Ptrs_[i]->id(), so
296  * the Id of the secondary atom in a pair is always greater than that of
297  * the primary atom. The total number of primary atoms in atom1Ptrs_ is
298  * nAtom1_. The total number of secondary atoms in Array atom2Ptrs_ (or
299  * the total number of distinct pairs) is nAtom2_. The total number of
300  * Atoms in the System in nAtom_.
301  *
302  * The only way legal way to loop over all atom pairs, using the public
303  * interface of a PairList, is to use a PairListIterator. See the
304  * documentation of PairListIterator for a discussion of its usage. The
305  * resulting loop over pairs is equivalent to the following double loop
306  * over private member variables of PairList:
307  *
308  * int i, j;
309  * Atom* atom1Ptr;
310  * Atom* atom2Ptr;
311  *
312  * \\ Loop over primary Atoms
313  * for (i = 0; i < nAtom1_; ++i) {
314  * atom1Ptr = atom1Ptrs_[i];
315  *
316  * // Loop over secondary atoms
317  * for (j = first_[i]; j < first_[i+1]; ++j) {
318  * atom2Ptr = atom2Ptrs_[j];
319  *
320  * // ... Do something with Atoms *atom1Ptr and *atom2Ptr.
321  *
322  * }
323  * }
324  *
325  * Note that DArray<int> first_ contains nAtom1_ + 1 elements. The first
326  * element, element first_[0] is equal to 0, and the last element,
327  * first_[nAtom1_], is always equal to nAtom2_, the total number of
328  * pairs.
329  *
330  * The number of primary Atoms nAtom1_ is generally less than nAtom_, the
331  * total number of atoms, because some Atoms have no neighbors with integer
332  * Ids higher than their own. This is always true, for example, for the
333  * Atom with the highest Id. Pointers to such atoms, which do not appear
334  * in the first nAtom1_ elements of atom1Ptrs_, are instead stored in the
335  * last elements of the atom1Ptrs_ array , in elements tList1 + 1, .... ,
336  * atomCapacity_ - 1. The previous position Vectors for these Atoms are
337  * stored in the corresponding elements of the oldPositions_ array.
338  * These pointers and previous positions retained because they are used
339  * in the isCurrent() method to calculate how far these Atoms have moved
340  * since the last time the PairList was built.
341  */
